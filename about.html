VolumeVisual

VolumeVisual is an educational tool that provides a friendly web interface for students to explore and interact with volume visualizations of a variety of data sets from real-world medical applications and scientific simulations. As the most popular techniques for volume visualization, direct volume rendering and isosurface rendering are utilized to allow users to draw important information from these data sets. Users learn how to compare and evaluate these two different visualization techniques in side-by-side panels, which can be interactively explored by various interactions. VolumeVisual is mainly developed with WebGL 2.0. To get better browser compatibility and performance, users are strongly recommended to download the latest version of Chrome, FireFox, or Edge and update Graphics Cards driver to the newest version.

Motivation

The motivation stems from the significance of direct volume rendering and isosurface rendering techniques in scientific and engineering applications. As two essential visual representation techniques, direct volume rendering and isosurface rendering have been widely used in different disciplines that require visualization of three-dimensional data sets. Many available volume visualization tools such as ParaView [1], VolView [2], and ImageVis3D [3] require much knowledge of visualization background and are too complicated for beginners in scientific visualization to use. This is why VolumeVisual is developed that incorporates simplicity, selectivity, interactivity, and efficiency in the design to help students effectively learn volume visualization knowledge.

Direct volume rendering

Direct volume rendering avoids the binary classification in isosurface rendering by using a GPU-accelerated raycasting method in [4] that includes ray traversal, reconstruction, classification (transfer function lookup), shading, and compositing in the fragment shader. Users can specify the transfer function, and map different value ranges to different colors and opacities. Furthermore, users are also provided with axis-aligned slicing planes to cut out the volume to better observe the inside structure. Users can scale and rotate the volume and examine from different viewpoints. Default lighting and viewing parameters are used to generate desired rendering results so that users can focus on the interaction. Users can also tweak the transfer function editor to get their preferred volume rendering results for each data set.

Isosurface rendering

Given a volumetric data set, we apply the isosurface similarity map method [5] [6] to identify representative surfaces. These surfaces are extracted using the march cube algorithm [7]. Users can switch between these surfaces with different representative isovalues, which enables users to get a comprehensive understanding of the volumetric data set. The Phong illumination model is used to render the output triangle primitives of isosurfaces. Both direct volume rendering and isosurface rendering share the same color map in the transfer function editor. Users can also use the same axis-aligned slicing planes in volume rendering to cut out the surface along three dimensions. For both methods, users can scale and rotate the volume and examine from different viewpoints. The same default lighting and viewing parameters are used to generate desired rendering results. One goal of VolumeVisual is to enable users to compare side-by-side, both direct volume rendering and isosurface rendering results generated from the same volumetric data set and under the same viewpoint. This would help them better grasp the pros and cons of each method.

Data sets

VolumeVisual comes with a set of example data sets from medical applications and scientific simulations. The data set description provides detailed information about each data set.

Interactions

VolumeVisual provides essential interaction functions to allow users to scale, rotate, and translate the volume and examine from different viewpoints. Users can use the mouse, touchpad, and keyboard to perform these interactions to get a good view of the volumetric data sets. VolumeVisual also provides sliders to perform the translation and scaling interactions. For direct volume rendering, users can specify the transfer function by mapping different value ranges to different colors and opacities. Users can slice the volume along the X, Y, and Z dimensions to cut out the volume in both views to better observe the inside structure.
